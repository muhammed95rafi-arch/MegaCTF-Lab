▶ THEORY

What is JWT?

JSON Web Token (JWT) is used for authentication. It has 3 parts separated by dots:

header.payload.signature

Structure

Header — algorithm type (HS256, RS256, none)
Payload — user data (user, role, exp)
Signature — verifies token integrity

alg:none Vulnerability

Some servers accept "alg":"none" in the header, meaning NO signature is required. An attacker can modify the payload freely!

# Original token header (decoded):
{"alg":"HS256","typ":"JWT"}

# Forged token header:
{"alg":"none","typ":"JWT"}

# Modified payload:
{"user":"admin","role":"admin"}

Attack Steps

1. Decode the original token (base64)
2. Change alg to none in header
3. Change role to admin in payload
4. Encode back to base64 (no signature needed)
5. Send the forged token
