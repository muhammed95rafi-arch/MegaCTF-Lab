▶ THEORY

Why Log Analysis Matters

Server logs record every action — logins, file access, errors, attacks. A SOC analyst reads logs to detect intrusions, find attacker IPs, and understand attack timelines.

Common Log Types

Access logs — HTTP requests to the server
Auth logs — login success/failure events
Error logs — application errors
System logs — OS-level events

What to look for

Multiple failed logins → Brute force
Unusual file access → Data theft
Suspicious uploads → Web shell
Admin access after failures → Compromise

Linux commands for log analysis

grep "FAIL" /var/log/auth.log
grep "10.0.0.5" access.log
awk '{print $1}' access.log | sort | uniq -c | sort -rn
cat app.log | grep -i "flag"
▶ QUESTIONS — Read the logs carefully!
Q1. What is the attacker's IP address?
Q2. How many failed login attempts were made before success?
Q3. What malicious file did the attacker try to upload?
Q4. What is the FLAG hidden in the logs?
