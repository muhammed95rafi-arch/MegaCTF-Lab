▶ THEORY

What is File Upload Vulnerability?

When a web server allows file uploads without proper validation, an attacker can upload a malicious script (web shell) and execute commands on the server.

How it works

A typical PHP web shell looks like:

<?php system($_GET['cmd']); ?>

If the server saves and executes this file, an attacker can run:

http://target.com/uploads/shell.php?cmd=whoami
http://target.com/uploads/shell.php?cmd=ls

Why servers are vulnerable

Many servers only check the file extension visually or rely on user-provided Content-Type, making it easy to bypass. A proper server must validate extension + MIME type + magic bytes.

Real-world impact

File upload vulnerabilities can lead to Remote Code Execution (RCE), full server takeover, data theft, and ransomware deployment.
